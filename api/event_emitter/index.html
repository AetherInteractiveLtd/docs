EventEmitter

EventEmitter

when(name?: string): EventListener

Parameters

name: string

The name of the event if there is one, refer to Using Blank Events otherwise.

Returns

EventListener

Returns a new EventListener.

Examples

import Tina, { EventEmitter } from '@rbxts/tina';

interface MyEvents {
    bar: [ newState: number ]; // (1)!
}

class Foo extends EventEmitter<MyEvents> /* (2)! */ {}
  1. This is the spread of arguments that any EventListeners for this specific event will receive.
  2. This will create an EventEmitter on our class which uses the event definitions from MyEvents.

Blank Events

You can use Blank Events when your EventEmitter is specifically mono-behaviour or if you've got a main (really overly important) event somewhere.

import Foo from `./Foo`;

const obj = new Foo(); // (1)!

obj.when("bar") // (2)!
    .do((state: number /* (3)! */) => {

    });
  1. EventEmitters can never be static, they need an object in order to work.
  2. We need to fetch a new EventListener for this event, .when(name) will deliver one.
  3. We defined this event as bar: [ newState: number ], so we can use this parameter in our first .do()
